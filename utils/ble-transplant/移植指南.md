# BLE 数据传输模块移植指南

## 一、概述

本指南用于将蓝牙连接、数据上传、数据压缩和数据传输功能移植到其他微信小程序项目中。

## 二、文件说明

### 2.1 核心文件

| 文件 | 功能 |
|------|------|
| [ble.ts](file:///c:\Users\OUYQ\Desktop\TRL\KML_BLE\miniprogram-2\miniprogram\utils\ble-transplant\ble.ts) | BLE 蓝牙底层 API 封装 |
| [kml-sender.ts](file:///c:\Users\OUYQ\Desktop\TRL\KML_BLE\miniprogram-2\miniprogram\utils\ble-transplant\kml-sender.ts) | KML/GPX 文件处理、压缩、数据分包发送 |

### 2.2 BLE 模块功能 (ble.ts)

- `openBluetoothAdapter()` - 初始化蓝牙适配器
- `closeBluetoothAdapter()` - 关闭蓝牙适配器
- `startBluetoothDevicesDiscovery()` - 开始搜索设备
- `stopBluetoothDevicesDiscovery()` - 停止搜索设备
- `createBLEConnection(deviceId)` - 连接 BLE 设备
- `closeBLEConnection(deviceId)` - 断开连接
- `getBLEDeviceServices(deviceId)` - 获取设备服务列表
- `getBLEDeviceCharacteristics(deviceId, serviceId)` - 获取特征值列表
- `findWritableCharacteristic(deviceId)` - 查找可写入特征值
- `findNotifyCharacteristic(deviceId)` - 查找可通知特征值
- `enableBLECharacteristicNotify(deviceId, serviceId, characteristicId, state)` - 启用通知
- `writeBLECharacteristicValue(deviceId, serviceId, characteristicId, value)` - 写入数据
- `onBLECharacteristicValueChange(callback)` - 监听数据接收

### 2.3 KML 发送模块功能 (kml-sender.ts)

**常量:**
- `BLE_CHUNK_SIZE = 230` - BLE 单次写入最大字节数
- `KML_END_MARKER = [0xFF, 0xFF, 0xFF]` - 发送结束符序列

**核心函数:**

| 函数 | 功能 |
|------|------|
| `calculateCRC32(buffer)` | 计算 CRC32 校验码 |
| `readFileAsArrayBuffer(filePath, compress, fileType)` | 读取文件并转换为 ArrayBuffer |
| `compressText(text)` / `compressTextByType(text, fileType)` | 压缩 KML/GPX 数据 |
| `decompress(compressedBuffer)` | 解压数据 |
| `chunkArrayBuffer(buffer, chunkSize)` | 将数据分块 |
| `getPayloadToSend(buffer, chunkSize)` | 生成发送载荷（含结束符） |
| `verifyChunkedPayload(buffer, chunkSize)` | 校验分包逻辑 |
| `sendKmlBufferToBle(buffer, options)` | 发送数据到 BLE 设备 |

**数据类型:**

```typescript
interface BLECharacteristic {
  uuid: string
  serviceId: string
}

interface SendProgress {
  sentBytes: number
  totalBytes: number
  percent: number
  currentChunk: number
  totalChunks: number
}

interface KmlSendOptions {
  deviceId: string
  chunkSize?: number      // 默认 230
  writeDelayMs?: number    // 默认 20ms
  onProgress?: (progress: SendProgress) => void
}
```

### 3.3 在页面中使用

**示例 - 蓝牙连接与数据传输:**

```typescript
import {
  openBluetoothAdapter,
  startBluetoothDevicesDiscovery,
  stopBluetoothDevicesDiscovery,
  createBLEConnection,
  findWritableCharacteristic,
  findNotifyCharacteristic,
  enableBLECharacteristicNotify,
  onBLECharacteristicValueChange,
  closeBLEConnection
} from '../../utils/ble-transplant/ble'

import {
  readFileAsArrayBuffer,
  sendKmlBufferToBle,
  calculateCRC32,
  compressText
} from '../../utils/ble-transplant/kml-sender'

Page({
  data: {
    devices: [],
    selectedDevice: null,
    status: 'idle' as 'idle' | 'scanning' | 'connected' | 'sending'
  },

  // 1. 初始化蓝牙
  async initBLE() {
    try {
      await openBluetoothAdapter()
      await startBluetoothDevicesDiscovery()
      this.setData({ status: 'scanning' })
      
      // 监听设备发现
      wx.onBluetoothDeviceFound((res) => {
        // 处理发现的设备
      })
    } catch (err) {
      console.error('蓝牙初始化失败:', err)
    }
  },

  // 2. 连接设备
  async connectDevice(deviceId: string) {
    try {
      await createBLEConnection(deviceId)
      
      // 查找可写入和可通知的特征值
      const writeChar = await findWritableCharacteristic(deviceId)
      const notifyChar = await findNotifyCharacteristic(deviceId)
      
      // 启用通知
      await enableBLECharacteristicNotify(
        deviceId, 
        notifyChar.serviceId, 
        notifyChar.uuid, 
        true
      )
      
      // 监听数据接收
      onBLECharacteristicValueChange((res) => {
        // 处理接收到的数据
      })
      
      this.setData({ status: 'connected' })
    } catch (err) {
      console.error('连接失败:', err)
    }
  },

  // 3. 选择并发送文件
  async chooseAndSendFile() {
    // 选择文件
    const res = await wx.chooseMessageFile({
      count: 1,
      type: 'file',
      extension: ['kml', 'gpx']
    })
    
    const file = res.tempFiles[0]
    const fileType = file.name.toLowerCase().endsWith('.gpx') ? 'gpx' : 'kml'
    
    // 读取并压缩文件（compress=true 启用压缩）
    const buffer = await readFileAsArrayBuffer(file.path, true, fileType)
    
    // 计算 CRC32
    const crc32 = calculateCRC32(buffer)
    console.log('CRC32:', crc32.toString(16).toUpperCase())
    
    // 发送到 BLE 设备
    await sendKmlBufferToBle(buffer, {
      deviceId: this.data.selectedDevice.deviceId,
      chunkSize: 230,        // BLE MTU 限制
      writeDelayMs: 20,       // 包间延迟
      onProgress: (progress) => {
        console.log(`发送进度: ${progress.percent}%`)
      }
    })
  }
})
```

## 四、注意事项

### 4.1 BLE 限制

- **MTU 限制**: 单次写入最大 20-512 字节，建议使用 230 字节以确保兼容性
- **分包发送**: 大文件需要分包发送，每包之间建议添加 20ms 延迟
- **结束符**: 发送完成后必须追加 `0xFF 0xFF 0xFF` 结束符序列

### 4.2 权限配置

在目标小程序的 `app.json` 中需要配置权限：

```json
{
  "permission": {
    "scope.userLocation": {
      "desc": "用于读取轨迹文件位置信息"
    },
    "scope.bluetooth": {
      "desc": "用于连接 BLE 设备传输数据"
    }
  },
  "requiredPrivateInfos": ["getLocation"]
}
```

### 4.3 微信开发者工具设置

1. 开启"调试基础库"详情中的"自动预览"
2. 确保手机与电脑在同一 WiFi 下
3. 在开发者工具中启用"蓝牙"调试

### 4.4 压缩算法说明

KML/GPX 数据压缩采用以下算法：

1. **解析文件**: 提取 Point/LineString 点数据
2. **转换为二进制**: 使用 int32 格式存储（经纬度 × 1,000,000）
3. **差分压缩**: 第一个点存储完整坐标，后续点存储与前一点的差值
4. **可变长度编码**: 使用 7-bit 可变长度编码进一步压缩

## 五、常见问题

### 5.1 连接失败

- 检查设备是否开启
- 确保蓝牙权限已授权
- 部分设备需要先配对才能连接

### 5.2 发送失败

- 检查设备是否支持 BLE 写入
- 确保 MTU 设置足够
- 尝试增加 `writeDelayMs` 延迟

### 5.3 数据校验

使用 `verifyChunkedPayload()` 函数可以在不连接设备的情况下验证分包逻辑是否正确。
