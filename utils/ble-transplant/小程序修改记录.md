# TRL-X 小程序修改记录

本文档详细记录了对 TRL-X 小程序的所有修改，包括新增功能、文件修改、技术实现和当前程序效果。

## 一、项目概述

### 1.1 项目目标
将 BLE（低功耗蓝牙）传输功能集成到 TRL-X 跑山赛小程序中，实现将比赛计划数据同步到智能手表设备的功能。

### 1.2 核心功能
- BLE 设备扫描、连接、订阅
- 比赛计划数据生成和打包
- 通过 BLE 分包发送数据到硬件设备
- 支持发送进度显示和错误处理

## 二、新增功能

### 2.1 BLE 蓝牙通信功能
- **设备扫描**：自动搜索附近的 BLE 设备
- **设备连接**：支持连接指定的 BLE 设备
- **服务发现**：自动发现设备的服务和特征值
- **数据订阅**：订阅设备的通知特征值，实现双向通信
- **数据发送**：支持向设备写入数据，分包发送大数据

### 2.2 计划数据同步功能
- **计划生成**：根据比赛信息生成完整的比赛计划
- **数据打包**：将计划数据打包为 JSON 格式
- **分包传输**：将大数据分包发送（230字节/包，间隔20ms）
- **进度显示**：实时显示发送进度百分比
- **错误处理**：完善的错误处理和重试机制

### 2.3 数据格式转换
- **TypeScript 到 JavaScript**：将 TypeScript 模块转换为 JavaScript，适配小程序环境
- **JSON 序列化**：将计划对象序列化为 JSON 字符串
- **UTF-8 编码**：将 JSON 字符串编码为 UTF-8 字节数组
- **ArrayBuffer 分块**：将大数据分块为适合 BLE 传输的小包

## 四、技术实现细节

### 4.1 BLE 通信流程

#### 4.1.1 设备连接流程
1. 初始化蓝牙适配器
2. 开始搜索设备
3. 用户选择设备
4. 建立连接
5. 获取设备服务
6. 获取特征值
7. 查找可写入和可通知特征值
8. 启用特征值通知

#### 4.1.2 数据发送流程
1. 检查设备连接状态
2. 生成计划数据（JSON 格式）
3. 转换为 UTF-8 字节数组
4. 分块处理（230 字节/包）
5. 依次发送每个数据包（间隔 20ms）
6. 发送结束符序列（`0xFF 0xFF 0xFF`）
7. 显示发送进度
8. 处理发送结果

### 4.2 数据格式

#### 4.2.1 BLE 载荷结构
```json
{
  "ver": 2,
  "raceDate": "2026-03-22",
  "raceName": "2026温岭黄金海岸跑山赛 - 25km",
  "segmentCount": 3,
  "segments": [
    {
      "segmentIndex": 1,
      "segmentName": "CP1 - 七彩小岛",
      "arrivalTimeBjt": "2026-03-22 11:49",
      "cutoffTimeBjt": "2026-03-22 08:50",
      "restMin": 5
    },
    {
      "segmentIndex": 2,
      "segmentName": "CP2 - 金沙滩",
      "arrivalTimeBjt": "2026-03-22 14:31",
      "cutoffTimeBjt": "2026-03-22 11:30",
      "restMin": 10
    },
    {
      "segmentIndex": 3,
      "segmentName": "终点",
      "arrivalTimeBjt": "2026-03-22 16:40",
      "cutoffTimeBjt": "",
      "restMin": 0
    }
  ]
}
```

#### 4.2.2 字段说明
- `ver`：版本号，固定为 2
- `raceDate`：比赛日期，格式 `YYYY-MM-DD`
- `raceName`：比赛名称
- `segmentCount`：分段数量
- `segments`：分段数组
  - `segmentIndex`：段序号（从 1 开始）
  - `segmentName`：段名称
  - `arrivalTimeBjt`：到达北京时间，格式 `YYYY-MM-DD HH:mm`
  - `cutoffTimeBjt`：关门北京时间，格式 `YYYY-MM-DD HH:mm`（终点可为空）
  - `restMin`：休息时间（分钟）

### 4.3 分包传输机制

#### 4.3.1 分包参数
- **分包大小**：230 字节/包
- **发送间隔**：20 毫秒
- **结束符**：`0xFF 0xFF 0xFF`（3 字节）

#### 4.3.2 发送示例
假设 JSON 数据为 526 字节：
- 第 1 包：0-229 字节（230 字节）
- 第 2 包：230-459 字节（230 字节）
- 第 3 包：460-525 字节（66 字节）
- 第 4 包：结束符 `0xFF 0xFF 0xFF`（3 字节）
- 总共 4 个包
